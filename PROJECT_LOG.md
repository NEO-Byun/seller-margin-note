# 프로젝트 로그

## 현재 산출물

- 공개 앱: https://neo-byun.github.io/seller-margin-note/
- GitHub repo: https://github.com/NEO-Byun/seller-margin-note
- 제품명: 셀러마진 수첩
- 현재 형태: 정적 웹앱. 판매가, 원가, 수수료, 배송비, 광고비, 반품률을 넣어 순이익 계산.

## 지금까지 결정

- 미국식 청구/미수금 도구는 폐기.
- 한국 타겟으로 전환.
- 커뮤니티 홍보는 사용자가 하기 싫어하므로 중심 전략에서 제외.
- standalone 웹사이트는 데모로 유지.
- 본 방향은 네이버 커머스솔루션마켓 입점형 솔루션으로 전환.

## 네이버 방향

제품 방향:

- 셀러마진 수첩 → 스마트스토어 마진 리포트

핵심 기능:

- 스마트스토어 상품 목록 기반 리포트
- 원가 CSV 업로드
- 상품별 예상 순이익 계산
- 팔수록 손해 상품 표시
- 목표 마진 판매가 제안
- 월 구독형 요금제 검토

## 확인된 네이버 조건

- 커머스솔루션마켓은 스마트스토어 판매자 대상 솔루션 마켓.
- 커머스API 기반 솔루션 제작 필요.
- 테스트 후 심사 과정을 거쳐 입점.
- 비즈월렛 결제와 정산 흐름 제공.
- 공식 문서 기준 마켓 입점 동의는 현재 법인 사업자만 가능.

## 사용자 확인 필요

- 법인 사업자 보유 여부.
- 스마트스토어 판매자 계정 보유 여부.
- 커머스API센터 가입 가능 여부.
- 네이버 입점 동의 진행 가능 여부.

## 다음 개발 작업

1. 현재 앱을 상품 1개 계산기에서 여러 상품 리포트로 전환.
2. CSV import 추가.
3. 손해/주의/양호 상품 분류.
4. 네이버 심사용 제품 설명서 작성.
5. 이후 서버 프로젝트 추가.

## 스크롤 이슈 메모

배포 앱 CSS에는 세로 스크롤을 막는 설정이 없음.

대화창에서 이전 내용이 보이지 않는 문제라면 Codex/ChatGPT 클라이언트의 긴 대화 표시 또는 context compaction 문제로 보임. 중요한 결정은 이 파일에 계속 누적.
